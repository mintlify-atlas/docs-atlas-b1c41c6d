## What You'll Build

In this quickstart, you'll:

1. Start a Cerbos PDP instance
2. Make an authorization check (it will fail - no policies yet!)
3. Create policies for an album-sharing application
4. See authorization decisions change as you add rules

By the end, you'll understand how Cerbos evaluates policies and makes authorization decisions.

<Note>
  **Prerequisites:** Docker installed, or download the [Cerbos binary](/installation/binary)
</Note>

## Step 1: Start Cerbos

Create a directory for your policies and start Cerbos:

<CodeGroup>
```bash Docker
# Create policy directory
mkdir -p cerbos-quickstart/policies

# Start Cerbos PDP
docker run --rm --name cerbos -d \
  -v $(pwd)/cerbos-quickstart/policies:/policies \
  -p 3592:3592 \
  -p 3593:3593 \
  ghcr.io/cerbos/cerbos:latest
```

```bash Binary
# Create policy directory
mkdir -p cerbos-quickstart/policies

# Create config file
cat > cerbos-quickstart/.cerbos.yaml <<EOF
---
server:
  httpListenAddr: ":3592"
storage:
  driver: "disk"
  disk:
    directory: policies
EOF

# Start Cerbos
cd cerbos-quickstart
./cerbos server --config=.cerbos.yaml
```
</CodeGroup>

<Info>
  **API Browser**: Open http://localhost:3592 in your browser to explore the Cerbos API with built-in documentation.
</Info>

```go Go SDK
package main

import (
	"context"
	"log"
	"github.com/cerbos/cerbos-sdk-go/cerbos"
)

func main() {
	c, err := cerbos.New("localhost:3593", cerbos.WithPlaintext())
	if err != nil {
		log.Fatalf("Failed to create client: %v", err)
	}

	principal := cerbos.NewPrincipal("bugs_bunny", "user")
	principal.WithAttr("beta_tester", true)

	r1 := cerbos.NewResource("album:object", "BUGS001")
	r1.WithAttributes(map[string]any{
		"owner":   "bugs_bunny",
		"public":  false,
		"flagged": false,
	})

	r2 := cerbos.NewResource("album:object", "DAFFY002")
	r2.WithAttributes(map[string]any{
		"owner":   "daffy_duck",
		"public":  true,
		"flagged": false,
	})

	batch := cerbos.NewResourceBatch()
	batch.Add(r1, "view:public", "comment")
	batch.Add(r2, "view:public", "comment")

	resp, err := c.CheckResources(context.Background(), principal, batch)
	if err != nil {
		log.Fatalf("Failed to check resources: %v", err)
	}
	log.Printf("%v", resp)
}
```

```python Python SDK
from cerbos.sdk.client import CerbosClient
from cerbos.sdk.model import Principal, Resource, ResourceList

with CerbosClient("http://localhost:3593", plaintext=True) as c:
    principal = Principal(
        "bugs_bunny",
        roles=["user"],
        attr={"beta_tester": True}
    )
    
    resources = ResourceList()
    resources.add(
        Resource(
            "BUGS001",
            "album:object",
            attr={
                "owner": "bugs_bunny",
                "public": False,
                "flagged": False
            }
        ),
        actions=["view:public", "comment"]
    )
    resources.add(
        Resource(
            "DAFFY002",
            "album:object",
            attr={
                "owner": "daffy_duck",
                "public": True,
                "flagged": False
            }
        ),
        actions=["view:public", "comment"]
    )
    
    resp = c.check_resources(principal, resources)
    print(resp)
```

## Step 3: Create Derived Roles

Let's define a derived role that assigns the `owner` role when a user owns a resource:

```bash Create derived_roles_common.yaml
cat > cerbos-quickstart/policies/derived_roles_common.yaml <<EOF
---
apiVersion: "api.cerbos.dev/v1"
derivedRoles:
  name: common_roles
  definitions:
    - name: owner
      parentRoles: ["user"]
      condition:
        match:
          expr: request.resource.attr.owner == request.principal.id
EOF
```

<Info>
  **Derived Roles** dynamically assign roles based on conditions. Here, users with the `user` role get the `owner` derived role when they own the resource.
</Info>

## Step 4: Create a Resource Policy

Now create a policy that gives owners full access to their albums:

```bash Create resource_album.yaml
cat > cerbos-quickstart/policies/resource_album.yaml <<EOF
---
apiVersion: api.cerbos.dev/v1
resourcePolicy:
  version: "default"
  importDerivedRoles:
    - common_roles
  resource: "album:object"
  rules:
    - actions: ['*']
      effect: EFFECT_ALLOW
      derivedRoles:
        - owner
EOF
```

Cerbos automatically detects the new policy files. Try the request again!

### Response: Owner Access Granted

```json
{
  "requestId": "quickstart",
  "results": [
    {
      "resource": {
        "id": "BUGS001",
        "kind": "album:object"
      },
      "actions": {
        "comment": "EFFECT_ALLOW",
        "view:public": "EFFECT_ALLOW"
      }
    },
    {
      "resource": {
        "id": "DAFFY002",
        "kind": "album:object"
      },

        "comment": "EFFECT_DENY",
```

</Check>

## Step 5: Add Conditional Access

Let's allow all users to view public albums by adding a conditional rule:

```bash Update resource_album.yaml
cat > cerbos-quickstart/policies/resource_album.yaml <<EOF
---
apiVersion: api.cerbos.dev/v1
resourcePolicy:
  version: "default"
  importDerivedRoles:
    - common_roles
  resource: "album:object"
  rules:
    - actions: ['*']
      effect: EFFECT_ALLOW
      derivedRoles:
        - owner

    - actions: ['view:public']
      effect: EFFECT_ALLOW
      roles:
        - user
      condition:
        match:
          expr: request.resource.attr.public == true
EOF
```

Try the request one more time:

### Response: Public Access Granted

```json
{
  "requestId": "quickstart",
  "results": [
    {
      "resource": {
        "id": "BUGS001",
        "kind": "album:object"
      },
      "actions": {
        "comment": "EFFECT_ALLOW",
        "view:public": "EFFECT_ALLOW"
      }
    },
    {
      "resource": {
        "id": "DAFFY002",
        "kind": "album:object"
      },
      "actions": {
        "comment": "EFFECT_DENY",
        "view:public": "EFFECT_ALLOW"
      }
    }
  ]
}
```

<Check>
  Now Bugs Bunny can view Daffy's **public** album, but still can't comment on it!
</Check>

## Challenge: Allow Comments on Public Albums

Can you figure out how to update the policy to allow users to comment on public albums?

<Accordion title="Solution">
  Add `comment` to the actions list in the public album rule:
  
  ```yaml
  - actions: ['view:public', 'comment']
    effect: EFFECT_ALLOW
    roles:
      - user
    condition:
      match:
        expr: request.resource.attr.public == true
  ```
</Accordion>
